# Claude Code Adapter

## 1. 适用场景

- 需要较强代码上下文理解、渐进式编辑、补测试和解释改动的仓库任务。
- 需要把 PRD 或 issue 改写为可执行开发任务。

## 2. 不适用场景

- 缺少仓库上下文的抽象建议。
- 用户要求一次性大范围重写而没有验收标准。

## 3. Prompt 构造注意事项

- 说明先读取哪些文件、如何确认现有架构和测试方式。
- 强调小步修改、保持现有风格、遵循现有依赖和工具链。
- 要求输出变更摘要和验证结果。

## 4. 必须包含的约束

- 不要重写无关模块。
- 不要引入新框架或大依赖，除非用户明确允许。
- 不要删除测试、弱化断言或隐藏错误。
- 不确定时列出待确认问题，不把猜测写成事实。

## 5. 推荐输出格式

```text
上下文理解：
实现计划：
修改文件：
测试/验证：
兼容性说明：
剩余风险：
```

## 6. 常见失败模式

- 把产品想法直接写成代码而没有验收标准。
- 没有先看现有模式，导致风格和架构不一致。
- 过早大范围重构。
