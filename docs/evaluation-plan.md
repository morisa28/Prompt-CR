# 评估计划

## 研究问题

RQ1：分级需求澄清是否能显著提升代码生成 Prompt 的完整性和可执行性？

RQ2：基于软件工程维度的 Prompt 评分能否有效预测 coding agent 执行任务的成功率或人工评价质量？

RQ3：错题本记录的高频缺失信息能否反向优化问题树，降低后续用户 Prompt 的关键字段缺失率？

## 实验 1：Prompt 完整度提升实验

对比对象：

- 用户原始 Prompt；
- 分级澄清后的结构化需求；
- 系统生成后的 coding agent Prompt。

指标：

- 总分提升；
- 关键字段缺失率降低；
- readiness level 变化；
- 8 个维度中的短板变化。

当前已实现：三阶段评分、动态追问、生成后自审、30 条 eval cases。

后续计划：邀请学生输入真实开发需求，统计原始 Prompt 到生成 Prompt 的平均提升。

## 实验 2：人工评分一致性实验

方法：

- 邀请 2-3 名具备编程经验的同学或老师；
- 使用同一 8 维评分表对 30 条 Prompt 打分；
- 比较系统评分与人工评分的一致性。

指标：

- 平均分差；
- 维度一致率；
- Spearman 相关系数。

当前状态：已准备 eval case 数据集和系统评分脚本。

## 实验 3：Coding Agent 执行效果实验

对比：

- 直接把原始 Prompt 交给 Codex；
- 把系统优化后的 Prompt 交给 Codex。

指标：

- 任务完成率；
- 是否出现无关修改；
- 是否运行验证命令；
- 是否说明未验证项；
- 人工可接受度。

当前状态：尚未接入真实 agent 执行实验，本项目不夸大该能力。

## 可量化验收指标

1. 完成 3 个核心场景的问题树、评分规则和 Prompt 模板。
2. 构建不少于 30 条 Prompt 质量评测样例。
3. 实现原始 Prompt、结构化需求、生成 Prompt 三阶段评分。
4. 实现错题本记录、统计和迭代建议。
5. 完成 Web 原型和 CLI 原型。
6. 原始 Prompt 到生成 Prompt 的平均评分提升不少于 25 分。
7. 关键字段缺失率降低不少于 40%。
8. 高质量 Prompt 样例中 80% 以上达到 usable 或 high_quality。
9. 输出不少于 4 份项目文档。
10. 完成一份可用于立项答辩的演示流程。
