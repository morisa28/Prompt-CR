# 开发计划

## 第 1 阶段：调研与需求分析

调研学生和初级开发者使用 coding agent 的常见失败场景，优先整理 bugfix、新功能、测试生成三类核心任务的 prompt 缺陷。

## 第 2 阶段：问题树与评分模型设计

建立 Level 0 到 Level 4 的分级问题树，设计 8 维 prompt 评分体系，确定每个问题与评分维度、最终 prompt 字段的绑定关系。

## 第 3 阶段：MVP 原型开发

实现 TypeScript 原型，包括需求会话、问题引擎、评分器、prompt 生成器、生成后 reviewer 和错题本模块。

## 第 4 阶段：示例库与测试集建设

建设 bugfix、feature、test generation 等核心示例，补充评分、动态追问、生成器、错题本和 Web API 测试，并建立不少于 30 条 eval cases。

## 第 5 阶段：实验评估与迭代

邀请学生使用系统生成 prompt，对比原始 prompt、结构化需求和生成 prompt 在任务清晰度、可执行性、验证完整度上的差异，统计低分维度和关键字段缺失率。

## 第 6 阶段：申报、结题材料与成果整理

整理项目文档、技术路线、创新点、实验数据、演示视频、用户反馈和可复现代码，形成大创申报与结题材料。
