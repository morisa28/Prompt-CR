# 大创申报书提纲

## 1. 项目名称

面向代码生成智能体的分级需求澄清与 Prompt 优化系统（CodePrompt Coach / Prompt-CR）

## 2. 项目背景

AI coding agent 已进入真实开发流程，但其效果高度依赖任务描述质量。学生和初级开发者常因需求表达不完整导致 agent 误解任务、扩大修改范围或跳过验证。本项目面向 AI coding agent 使用中需求表达不清、上下文缺失、约束不足和验收标准模糊的问题，构建一个需求澄清、质量评分、Prompt 生成、生成后审查和错题本持续优化的闭环系统。

## 3. 研究意义

本项目把 prompt 编写转化为需求工程训练问题，提升用户的软件工程表达能力和 coding agent 的任务执行质量。

## 4. 国内外现状简析

现有工具多聚焦模型能力、IDE 集成或代码补全，较少系统化训练用户如何描述开发任务、约束和验收标准。Prompt 工程资料多为经验总结，缺少面向代码生成任务的评分和反馈闭环。

## 5. 主要研究内容

- 核心三场景分类：Bug 修复、新功能开发、测试生成。
- 分级需求澄清问题树。
- Prompt 质量评分模型。
- 目标工具 prompt 生成器。
- 生成后自审机制。
- 错题本与持续优化机制。

## 6. 拟解决的关键问题

- 模糊需求如何转成结构化开发任务。
- 如何量化 prompt 是否适合 coding agent。
- 如何避免 agent 无关重构、漏测和虚构结果。
- 如何把失败经验沉淀为可迭代数据。

## 7. 技术路线

采用 TypeScript 轻量原型：动态问题树 -> 结构化需求 -> 反作弊评分器 -> 生成器 -> reviewer -> lesson engine -> Web 教导式交互。

## 8. 创新点

见 `docs/innovation-points.md`，重点包括分级提问、量化评分、双阶段评估和错题本驱动迭代。

## 9. 可行性分析

原仓库已有软件工程分支、adapter、eval 和 lessons 资产；MVP 不依赖复杂模型训练，可先用规则化结构实现闭环，适合大创周期内完成。

## 10. 阶段计划

按调研、设计、原型、示例测试、实验迭代、结题材料六阶段推进。

## 11. 预期成果

- 可运行 MVP 原型。
- 软件工程 prompt 问题树和评分规则。
- 至少 4 个完整示例。
- 大创申报文档与技术路线材料。
- prompt 失败模式与错题本样例。

## 12. 风险与对策

- 风险：评分规则过于粗糙。对策：通过示例和学生实验迭代权重。
- 风险：问题过多影响体验。对策：采用分级提问和阻塞问题优先策略。
- 风险：生成 prompt 仍缺少上下文。对策：生成后自审并写入错题本。

## 13. 团队分工建议

- 项目负责人：总体设计、实验组织、申报材料。
- 工程开发：TypeScript 原型、CLI/Web UI、测试。
- 评测设计：评分规则、eval cases、实验数据分析。
- 文档与示例：示例库、用户手册、教学材料。

## 14. 经费使用建议

主要用于云端测试环境、模型 API 调研额度、问卷和用户实验激励、演示材料制作和论文/竞赛材料整理。

## 15. 结题验收指标

- 能演示完整闭环。
- MVP 聚焦 3 个核心场景，扩展场景折叠保留。
- 至少 30 条 Prompt 质量 eval cases。
- 评分报告包含 8 个维度。
- 生成 prompt 包含工作目录、目标、上下文、约束、验证和报告格式。
- 错题本能落盘、统计低分维度并生成迭代建议。
