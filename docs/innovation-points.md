# 创新点

1. **从“AI 代替用户理解需求”转向“AI 引导用户理解需求”**：系统不直接猜测用户目标，而是通过分级问题帮助用户显性化任务。
2. **面向代码生成任务的分级提问机制**：围绕软件工程场景拆分 Level 0 到 Level 4，逐步收集任务类型、交付物、上下文、约束和验证。
3. **Prompt 质量量化评分**：用 8 个维度把 prompt 质量转为可解释分数，帮助用户知道哪里缺失。
4. **生成前与生成后双阶段评估**：既评估原始需求，也评估生成后的最终 prompt，避免生成器自身遗漏关键维度。
5. **错题本驱动持续优化**：低分维度自动转为 `PromptMistake`，反向指导问题树、评分规则和模板迭代。
6. **面向 coding agent 的软件工程约束模板**：强调先读文件、最小修改、禁止无关重构、验证命令和失败报告。
7. **兼具工具属性和教学属性**：既能生成可用 prompt，也能训练学生表达需求、约束、验收和风险。
