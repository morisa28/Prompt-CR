# 风险分析

## 评分规则偏主观

当前评分器是规则型 MVP，可能无法覆盖所有真实开发语境。对策是建立 eval cases，并在后续引入人工评分一致性实验校准权重。

## 用户回答仍然不完整

动态追问只能降低缺失概率，不能保证用户一定提供高质量信息。对策是在 Web 中显示追问原因、示例答案和低分维度，让用户理解为什么需要补充。

## 错题本被误解为自动学习

当前错题本不会自动修改问题树或评分规则。对策是在文档和答辩中明确其定位是半自动迭代建议机制。

## Web 原型复杂度上升

教导式交互比一次性表单复杂，可能带来维护成本。对策是保持后端 API 简洁：`evaluate`、`follow-up`、`lessons` 三类入口。

## Node 环境复现问题

原项目依赖 Node 24 strip-types，不利于复现。本轮改为 Node 20 + tsx + TypeScript，降低老师和同学运行门槛。
