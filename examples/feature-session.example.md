# Feature Development 场景完整示例

## 原始需求

```text
帮我做一个课程收藏功能。
```

## 分级问答

| Level | 问题 | 回答 |
|---:|---|---|
| 0 | 任务类型 | 新功能开发 |
| 1 | 最终结果 | 学生可以在课程卡片和详情页收藏/取消收藏课程 |
| 1 | 交付物 | Codex 直接修改代码，并输出变更报告 |
| 2 | 工作目录 | `F:/workspace/course-app` |
| 2 | 技术栈 | React + TypeScript + Vite |
| 2 | 相关文件 | `src/pages/CourseList.tsx`、`src/pages/CourseDetail.tsx`、`src/store/user.ts` |
| 3 | 修改范围 | 可新增收藏状态管理和本地持久化，不改后端接口 |
| 3 | 非目标 | 不实现云同步，不改登录流程，不改课程搜索 |
| 4 | 验证方式 | `npm run build`，手动验证列表页与详情页收藏状态一致 |

## 最终 Prompt

```text
你是 Codex，请在 F:/workspace/course-app 实现课程收藏功能。

任务目标：
学生可以在课程卡片和课程详情页收藏或取消收藏课程，列表页和详情页状态保持一致。

项目背景：
React + TypeScript + Vite。当前不改后端接口，优先使用现有状态管理或本地持久化方式。

相关文件：
- src/pages/CourseList.tsx
- src/pages/CourseDetail.tsx
- src/store/user.ts

执行步骤：
1. 先读取页面、状态管理、路由和现有组件风格。
2. 确认课程 ID、课程卡片和详情页的数据来源。
3. 给出简短实现计划。
4. 在必要范围内实现收藏/取消收藏、状态同步和持久化。
5. 覆盖加载、空列表和重复点击边界。
6. 运行 `npm run build`，并说明手动验证步骤。

硬性约束：
- 不要实现云同步。
- 不要修改登录流程。
- 不要修改课程搜索。
- 不要新增依赖，除非明确必要并说明原因。
- 不要全仓库格式化。

最终输出：
- 实现思路
- 修改文件
- 关键逻辑
- 用户流程覆盖
- 验证命令与结果
- 剩余风险
```
