# CodePrompt Coach Checklists

## 1. 分级需求澄清检查表

- [ ] Level 0 是否识别任务类型？
- [ ] Level 1 是否明确目标与交付物？
- [ ] Level 2 是否收集工作目录、技术栈、相关文件、日志或命令？
- [ ] Level 3 是否确认修改范围、禁止事项、依赖和兼容边界？
- [ ] Level 4 是否确认验证命令、手动验收和最终报告格式？
- [ ] 每个问题是否绑定评分维度？
- [ ] 每个问题是否绑定最终 prompt 字段？
- [ ] 阻塞问题是否没有被静默跳过？

## 2. Prompt 评分检查表

- [ ] 是否给出总分？
- [ ] 是否给出 8 个维度得分？
- [ ] 是否说明扣分原因？
- [ ] 是否列出优先改进建议？
- [ ] 是否判断适不适合交给 coding agent？
- [ ] 是否列出最缺失的 3 个关键信息？
- [ ] 评分是否区分原始 prompt、结构化需求和生成后 prompt？

## 3. Coding Agent Prompt 检查表

- [ ] 是否明确目标工具？
- [ ] 是否明确工作目录？
- [ ] 是否说明先读取哪些文件或如何发现相关文件？
- [ ] 是否要求先分析根因或方案再修改？
- [ ] 是否明确允许修改和禁止修改的范围？
- [ ] 是否禁止无关重构、全仓库格式化、删除测试和降低断言？
- [ ] 是否列出验证命令或要求从配置识别验证命令？
- [ ] 是否要求失败时记录错误、说明阻塞原因和下一步？
- [ ] 最终报告是否包含修改文件、关键改动、验证结果和剩余风险？

## 4. Bugfix Prompt 检查表

- [ ] 是否包含错误日志或明确要求先收集日志？
- [ ] 是否包含复现步骤或失败命令？
- [ ] 是否要求先复现失败？
- [ ] 是否要求根因证据绑定文件、日志或命令？
- [ ] 是否禁止删除测试、降低断言、注释失败逻辑或吞掉异常？
- [ ] 是否验证原失败路径？

## 5. Feature Prompt 检查表

- [ ] 是否描述用户可观察的新行为？
- [ ] 是否包含用户流程？
- [ ] 是否列出 MVP 范围和非目标？
- [ ] 是否保护现有 API、路由、数据格式和用户功能？
- [ ] 是否覆盖加载、错误、空状态、权限或边界场景？
- [ ] 是否包含可测试验收标准？

## 6. Test Generation Prompt 检查表

- [ ] 是否明确被测对象？
- [ ] 是否列出需覆盖行为？
- [ ] 是否说明测试层级和测试框架？
- [ ] 是否读取现有测试风格？
- [ ] 是否禁止表面测试、跳过测试和弱化断言？
- [ ] 是否要求运行测试命令并报告结果？

## 7. Code Review Prompt 检查表

- [ ] 是否明确审查范围？
- [ ] 是否以 findings 开头？
- [ ] 是否按严重程度排序？
- [ ] 每条 finding 是否有文件位置？
- [ ] 每条 finding 是否有影响和修复建议？
- [ ] 是否列出 open questions、test gaps 和 residual risk？

## 8. Repository Analysis Prompt 检查表

- [ ] 是否明确只读？
- [ ] 是否要求读取 README、配置、入口和顶层目录？
- [ ] 是否输出文件树和模块表？
- [ ] 技术栈、入口和命令是否绑定文件证据？
- [ ] 是否区分事实、推断和未读取区域？
- [ ] 是否禁止修改文件和输出 secrets？

## 9. Prompt 自审检查表

- [ ] 生成后是否再次评分？
- [ ] 是否识别低分维度？
- [ ] 是否给出修订建议？
- [ ] 是否判断是否通过最低阈值？
- [ ] 是否在低分时创建 PromptMistake？

## 10. 错题本检查表

- [ ] 是否记录原始 prompt？
- [ ] 是否记录生成 prompt？
- [ ] 是否记录生成前后得分？
- [ ] 是否记录低分维度？
- [ ] 是否记录 mistakeType、mistakeDescription、suggestedFix？
- [ ] 是否标记 shouldUpdateQuestionTree？
- [ ] 是否标记 shouldUpdateScoringRule？
- [ ] 是否标记 shouldUpdatePromptTemplate？
- [ ] 是否避免保存敏感个人信息、密钥或完整私人对话？

## 11. 最终交付前检查表

- [ ] README 是否体现 CodePrompt Coach 新定位？
- [ ] 非软件工程领域是否已归档或排除在 MVP 外？
- [ ] 是否存在问题树、评分规则、prompt 模板、自审和 lessons 机制？
- [ ] 是否有至少 4 个完整示例？
- [ ] 是否有大创申报材料？
- [ ] 是否能运行 demo 或测试？
