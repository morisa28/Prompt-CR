# Security Boundary

## 1. 适用场景

- 防御性安全审查、威胁建模、日志分析、权限审计、安全加固、漏洞修复建议。
- RAG 权限控制和内部知识库访问边界。

## 2. 禁止输出

- 不提供攻击性利用链、payload、绕过权限、隐蔽攻击、恶意持久化、规避检测或数据窃取步骤。
- 不帮助访问未授权系统。
- 不输出真实 secrets 或指导绕过授权。

## 3. 允许输出

- 威胁模型。
- 安全需求和权限边界。
- 防御性日志分析。
- 加固建议。
- 安全测试计划。
- 修复优先级和验证方式。

## 4. 必须提醒

- 只支持合法授权环境下的防御性工作。
- 对生产系统、凭证、隐私数据和破坏性测试必须要求明确授权与回滚方案。

## 5. 缺失输入

- 授权范围。
- 系统边界。
- 资产、数据分类、用户角色、日志来源。
- 允许和禁止的测试方式。

## 6. 推荐安全 Prompt 结构

```text
请仅在授权范围内做防御性安全分析。
输出资产、边界、威胁、风险、修复建议、验证方法和剩余风险。
禁止攻击链、payload、绕过权限、隐蔽攻击或未授权访问。
```
